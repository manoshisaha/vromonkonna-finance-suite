Host payment tiers
How host payment works

Every trip has exactly one Host Budget, no matter how many hosts are assigned to it.

Domestic trips, one host: the Host Budget is set entirely by that host's own tier — Beginner is a fixed amount (optionally floored/capped by its own Minimum/Maximum), Intermediate and Advanced are a percentage of Adjusted Profit with a minimum floor and optional maximum ceiling.

Domestic trips, 2+ hosts: if anyone on the team (any role) is Intermediate or Advanced, the budget instead uses the Advanced tier's own rate as a ceiling — the Lead's own tier no longer matters for the budget size in this case. An all-Beginner team still falls back to using the (single) Lead's own tier, same as the one-host case.

Foreign trips: the Host Budget comes from the fee set in Settings → Foreign trip defaults instead — no tiers involved at all.

Splitting it (two stages): the budget is first divided by role weight (below) — or by a custom "Weight override" typed in on New Trip for that specific trip, if you want a different split than the default (e.g. two hosts who shared the work exactly evenly). Then each host's own slice is individually floored/capped to their own tier's Minimum/Maximum. A minimum is always honored — even if that pushes the total paid above the original budget — but a maximum still caps a share downward. So the amount actually paid to hosts can differ slightly from the Host Budget number itself; that's expected, not a bug.

Foreign trips: the Host Budget is a fixed base amount plus a per-participant rate instead — set per trip in New Trip, not from the tiers below.

Once the total is set, it's split among every assigned host according to their Role Weight (Lead / Co-host / Support), configured just below.
These feed directly into the calculation engine used on Dashboard, New Trip, Trip History, Reports, and Funds.

Multi-host role weights
